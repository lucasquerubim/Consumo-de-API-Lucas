# 🚀 PokéDex Mobile - Consumo de API

Este projeto foi desenvolvido como parte do **Desafio 01** da disciplina de Desenvolvimento e Integração de Sistemas Móveis. O objetivo é criar uma aplicação que consome a [PokéAPI](https://pokeapi.co/) para exibir informações sobre Pokémons de forma responsiva.

## 📱 Demonstração
- **Deploy:** [Insira o link do Netlify/GitHub Pages aqui]
- **Visualização:** Focada em dispositivos móveis (Mobile First).

## 🛠️ Tecnologias Utilizadas
* **HTML5:** Estrutura semântica.
* **CSS3:** Estilização com foco em Flexbox e Media Queries para responsividade.
* **JavaScript (ES6+):** Manipulação de DOM e consumo de API assíncrona com `fetch`.
* **PokéAPI:** Fonte de dados dos Pokémons.

## ⚙️ Funcionalidades
- [ ] Busca de Pokémon por nome ou ID.
- [ ] Exibição de imagem, tipos, altura e peso.
- [ ] Layout adaptável para diferentes tamanhos de tela.
- [ ] (Opcional) Listagem inicial dos primeiros 20 Pokémons.
